Module Development Guidelines

CSS Structure

  • Use BEM naming: .yourmodule, .yourmodule__element, .yourmodule--modifier
  • Root container: Always use class="tonika-module yourmodule yourmodule--card"
  • Use global components: .tonika-btn, .tonika-input, .tonika-select
  • Use CSS variables: var(--color-bg-primary), var(--spacing-md), etc.

JavaScript Integration

  • Mount your module with: new YourModule({ mount: "#element-id" })
  • Generate HTML with proper BEM classes
  • Use global CSS classes for common elements
  • Provide a destroy() method for cleanup

Testing Checklist

  • ✅ Test in both light and dark themes
  • ✅ Ensure responsive behavior on mobile
  • ✅ Verify accessibility with screen readers
  • ✅ Test alongside other modules (layout compatibility)
  • ✅ Check browser console for errors
